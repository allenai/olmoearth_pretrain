## Landsat converter incompatibility

The rslearn output from the EC2 Landsat pipeline uses layer naming `landsat`, `landsat.1`, ..., `landsat.11` (12 temporal groups).

The existing conversion code expects a different layout:
- `landsat.py` → `convert_monthly()` looks for `landsat_mo01` through `landsat_mo12`
- `convert_temporal_stack()` expects one layer with 12 `group_idx` values, not 12 separate dirs
- `convert_all.py` doesn't register `convert_landsat` in `CONVERTERS` / `ALL_MODALITIES` at all

**Band structure is fine** — B8 separate from B1-B7,B9-B11 matches `Modality.LANDSAT` in `constants.py`.

2. New converter that iterates `landsat`, `landsat.1`, ..., `landsat.11` directly
